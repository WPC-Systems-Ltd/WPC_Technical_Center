# `No wifi connection` on PYCO WIFI Series Screen

### Full functionality, no need to worry!

The PYCO WIFI series is designed as a **highly independent** device. Even if the screen displays `No wifi connection`, it only means that the **development board is not currently connected to a wireless network**. This **does not affect** the core functions of the development board:

1.  **Independent Operation:**
    - Even without a Wi-Fi connection, **all primary functions** of the product (e.g., data acquisition, I/O control, execution of built-in programs) will **operate normally**.
    - The development board will continue to execute your custom code or default functions independently.

2.  **When is Wi-Fi needed?**
    - Wi-Fi connectivity is only required when you need functions such as **cloud data upload**, **remote control**, or **configuration via the web interface**.

3.  **How to clear the message:**
    - If you wish to remove this message, you can connect the development board to a Wi-Fi network via code, or choose to **turn off the screen display** or **modify the display content** within your application.
