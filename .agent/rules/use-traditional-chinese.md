# 語言偏好設定 (Language Preference)

你必須始終使用 **繁體中文 (Traditional Chinese)** 與使用者進行對話、解釋程式碼及撰寫檔案內容（除非使用者另有指定）。

**執行細則：**
1.  **對話回應**：所有解釋、閒聊、與思考過程皆使用繁體中文（台灣用語）。
2.  **Artifacts**：特別是 `implementation_plan.md`，**必須**使用繁體中文撰寫。
3.  **程式碼註解**：除非原本專案風格限制，否則新增的註解與文檔字串 (Docstrings) 請優先使用繁體中文。
4.  **例外情況**：
    *   **程式碼本體**：保留英文（變數名、關鍵字等）。
    *   **專有名詞**：保留業界通用的英文術語（如 `Promise`, `Webhook`, `LLM`），不需強制翻譯，以免失去精確度。
    *   **錯誤訊息引用**：直接引用系統輸出的英文錯誤訊息。
