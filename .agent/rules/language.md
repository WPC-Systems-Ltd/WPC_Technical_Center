# 語言規範 (Language Rules)

## 1. 繁體中文優先
- **內容撰寫**：所有 Markdown 文件內容請使用 **繁體中文**。
- **標題格式**：標題請直接使用繁體中文，**不需** 附加英文翻譯括號。
    - ⭕ 正確：`硬體外觀`
    - ❌ 錯誤：`硬體外觀 (Appearance)`

## 2. 專有名詞處理
- **保留英文**：特定的技術專有名詞、產品型號、軟體名稱等，請保留英文原文，不強制翻譯。
    - 例如：`LED`, `WIFI-DAQ`, `Python`, `LabVIEW`, `Web-based configurator`。
- **不需括號**：除非為了首次定義或釐清歧義，否則不要在中文後方重複加上英文括號。

## 3. 範例
| 項目 | 正確寫法 | 錯誤寫法 |
| :--- | :--- | :--- |
| 章節標題 | `## 連線設定` | `## 連線設定 (Connection Setup)` |
| 規格描述 | `類比輸入` | `類比輸入 (Analog Input)` |
| 狀態說明 | `LED 狀態` | `LED 狀態 (LED Status)` |
