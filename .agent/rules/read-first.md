# 檔案修改安全守則

嚴禁在未先驗證當前內容的情況下修改任何檔案。

**強制執行程序：**
1.  **先讀取 (Read First)**：在產生任何寫入指令之前，**務必**使用 `view_file`（若為針對性檢查則用 `grep_search`）來檢視你打算編輯的目標檔案。
2.  **確認狀態 (Verify State)**：確認你瞄準的程式碼內容與你的預期完全相符。
3.  **注意上下文 (Context Awareness)**：理解周圍的程式碼行，以確保你的編輯在語法和邏輯上都是正確的。

**核心理由：**
盲目的編輯會導致持續性的錯誤、邏輯被覆蓋以及建置失敗。請始終將檔案系統視為唯一的真理來源。
